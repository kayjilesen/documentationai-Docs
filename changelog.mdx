## Recente updates

<Callout kind="info">
  Bekijk deze changelog om op de hoogte te blijven van nieuwe features, bugfixes en verbeteringen aan de documentatieruimte. We brengen regelmatig updates uit om je ervaring met het beheren van projectdocumentatie te verbeteren.
</Callout>

<Update label="2024-10-15" description="v1.2.0" tags={["feature", "improvement"]}>

## Nieuwe features

- Ondersteuning toegevoegd voor aangepaste merkkleuren in documentatiethema's, te beginnen met `#1A2D35` voor Kay Jilesen-branding.
- Mermaid-diagramweergave geïntroduceerd voor het visualiseren van webdesign-workflows en projectstrategieën.

## Verbeteringen

- Zoekfunctionaliteit verbeterd voor betere indexering van WordPress-specialistengidsen en casestudies.
- Laadtijden met `>30%` geoptimaliseerd voor documentatiepagina's met zware afbeeldingsmockups.

## Bugfixes

- Weergaveproblemen met Nederlandstalige content in dienstbeschrijvingen opgelost.
- Ankerlink-navigatie hersteld voor secties zoals "Strategie.Design.Code."

</Update>

<Update label="2024-09-20" description="v1.1.0" tags={["feature", "bugfix"]}>

## Nieuwe features

- Uitklapbare FAQ-secties geïntegreerd voor veelgestelde vragen over freelance webdesign- en developmentdiensten.
- Card-gebaseerde showcases toegevoegd voor klantcases zoals Olli, Ruud Lenssen en VicTech.

## Bugfixes

- Laden van afbeeldingen gecorrigeerd voor mockups vanuit `https://new.kayjilesen.dev/wp-content`.
- Responsive design-problemen op mobiel voor portfolio-galerijen verholpen.

## Verbeteringen

- Navigatie naar belangrijke pagina's zoals "Diensten" en "Kom in contact" gestroomlijnd.

</Update>

<Update label="2024-08-10" description="v1.0.0" tags={["breaking", "feature"]}>

## Eerste release

- Kerndocumentatieplatform gelanceerd met secties voor Strategie, Design en Code.
- Starttemplates toegevoegd voor het organiseren van projectdocumentatie, diensten en casestudies.

## Breaking changes

- Gemigreerd van legacy Markdown naar volledige MDX-ondersteuning voor interactieve componenten.

</Update>

## Hoe te upgraden

Volg deze stappen om je documentatie actueel te houden.

<Steps>
  <Step title="Versie controleren" icon="package">
    Controleer je huidige versie door het volgende uit te voeren:

    ```
    npm list kay-jilesen-docs
    ```
  </Step>
  <Step title="Pakket bijwerken" icon="download">
    <Tabs>
      <Tab title="npm" icon="package">
        ```
        npm update kay-jilesen-docs
        ```
      </Tab>
      <Tab title="yarn" icon="package">
        ```
        yarn upgrade kay-jilesen-docs
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Wijzigingen bekijken" icon="book-open">
    Scan deze changelog op breaking changes en test je pagina's.
  </Step>
</Steps>

<Callout kind="tip">
  Abonneer je op notificaties of volg de repository om meldingen te ontvangen bij nieuwe releases. Draag bij door functies voor te stellen die zijn afgestemd op webdesign- en WordPress-workflows.
</Callout>
