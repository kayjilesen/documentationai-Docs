## Webhook Setup

Set up webhooks to receive real-time notifications about documentation events like page updates or new versions.

<Steps>
  <Step title="Create Webhook" icon="settings">
    Navigate to your project settings and select the `Integrations` tab.
  </Step>
  <Step title="Add Endpoint URL" icon="link">
    Enter your webhook URL, such as `https://your-webhook-url.com/kayjilesen`.
  </Step>
  <Step title="Select Events" icon="bell">
    Choose events like `page.updated` or `version.published`.
  </Step>
  <Step title="Test and Save" icon="check-circle">
    Send a test payload and verify receipt.
  </Step>
</Steps>

Here is an example webhook payload:

<CodeGroup tabs="JSON">
  ```json
  {
    "event": "page.updated",
    "project": "kay-jilesen-docs",
    "page": {
      "id": "integrations",
      "path": "/integrations"
    },
    "timestamp": "2024-10-15T10:30:00Z"
  }
  ```
</CodeGroup>

## API Access for Custom Extensions

Use the Kay Jilesen API to build custom integrations. Authenticate with your API key.

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Bearer YOUR_API_KEY`
</ParamField>

<ParamField query="project" param-type="string" required="false">
  Project identifier, e.g., `kay-jilesen-docs`.
</ParamField>

Example API call to list pages:

<Tabs>
  <Tab title="JavaScript" icon="code">
    ```javascript
    const response = await fetch('https://api.example.com/v1/projects/kay-jilesen-docs/pages', {
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY'
      }
    });
    const pages = await response.json();
    console.log(pages);
    ```
  </Tab>
  <Tab title="Python" icon="python">
    ```python
    import requests
    response = requests.get(
      'https://api.example.com/v1/projects/kay-jilesen-docs/pages',
      headers={'Authorization': 'Bearer YOUR_API_KEY'}
    )
    pages = response.json()
    print(pages)
    ```
  </Tab>
</Tabs>

<ResponseField name="pages" field-type="array" required="true">
  Array of page objects with `id`, `path`, and `updated_at`.
</ResponseField>

<Callout kind="alert">
Never expose your `YOUR_API_KEY` in client-side code. Use server-side proxies for security.
</Callout>

## Importing from External Sources

<Expandable title="Import from GitHub or CMS" default-open="true">
Bring in existing documentation from GitHub repos, Notion, or WordPress exports.

1. Connect your repo in settings.
2. Select branches and paths.
3. Map Markdown files to pages.

### CMS Import
Supports CSV, JSON, or Markdown bulk uploads. Use the API endpoint `POST /v1/import` with a file payload.
