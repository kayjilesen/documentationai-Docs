## Overzicht

Integreer je documentatie met populaire diensten van derden om je workflow te stroomlijnen. Automatiseer updates vanuit GitHub-repositories, ontvang Slack-notificaties bij wijzigingen, stel webhooks in voor realtime meldingen en gebruik de API voor aangepaste extensies. Je kunt ook content importeren uit externe bronnen zoals Markdown-bestanden of CMS-exports.

<Callout kind="tip">
Begin met de GitHub-integratie voor automatische documentatiesynchronisatie vanuit je repository.
</Callout>

## Webhook instellen

Stel webhooks in om realtime meldingen te ontvangen over documentatiegebeurtenissen zoals pagina-updates of nieuwe versies.

<Steps>
  <Step title="Webhook aanmaken" icon="settings">
    Ga naar je projectinstellingen en selecteer het tabblad `Integraties`.
  </Step>
  <Step title="Endpoint-URL toevoegen" icon="link">
    Voer je webhook-URL in, bijvoorbeeld `https://your-webhook-url.com/kayjilesen`.
  </Step>
  <Step title="Gebeurtenissen selecteren" icon="bell">
    Kies gebeurtenissen zoals `page.updated` of `version.published`.
  </Step>
  <Step title="Testen en opslaan" icon="check-circle">
    Verstuur een test-payload en controleer de ontvangst.
  </Step>
</Steps>

Hier is een voorbeeld van een webhook-payload:

<CodeGroup tabs="JSON">
  ```json
  {
    "event": "page.updated",
    "project": "kay-jilesen-docs",
    "page": {
      "id": "integrations",
      "path": "/integrations"
    },
    "timestamp": "2024-10-15T10:30:00Z"
  }
  ```
</CodeGroup>

## API-toegang voor aangepaste extensies

Gebruik de API om aangepaste integraties te bouwen. Authenticeer met je API-sleutel.

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Bearer YOUR_API_KEY`
</ParamField>

<ParamField query="project" param-type="string" required="false">
  Project-ID, bijv. `kay-jilesen-docs`.
</ParamField>

Voorbeeld API-aanroep om pagina's op te halen:

<Tabs>
  <Tab title="JavaScript" icon="code">
    ```javascript
    const response = await fetch('https://api.example.com/v1/projects/kay-jilesen-docs/pages', {
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY'
      }
    });
    const pages = await response.json();
    console.log(pages);
    ```
  </Tab>
  <Tab title="Python" icon="python">
    ```python
    import requests
    response = requests.get(
      'https://api.example.com/v1/projects/kay-jilesen-docs/pages',
      headers={'Authorization': 'Bearer YOUR_API_KEY'}
    )
    pages = response.json()
    print(pages)
    ```
  </Tab>
</Tabs>

<ResponseField name="pages" field-type="array" required="true">
  Array van pagina-objecten met `id`, `path` en `updated_at`.
</ResponseField>

<Callout kind="alert">
Stel je `YOUR_API_KEY` nooit bloot in client-side code. Gebruik server-side proxies voor beveiliging.
</Callout>

## Importeren vanuit externe bronnen

<Expandable title="Importeren vanuit GitHub of CMS" default-open="true">
Haal bestaande documentatie op uit GitHub-repositories, Notion of WordPress-exports.

### GitHub importeren
1. Verbind je repository in de instellingen.
2. Selecteer branches en paden.
3. Koppel Markdown-bestanden aan pagina's.

### CMS importeren
Ondersteunt CSV-, JSON- of Markdown-bulkuploads. Gebruik het API-endpoint `POST /v1/import` met een bestandspayload.
</Expandable>

<Expandable title="Geavanceerd: aangepaste parsers">
Voor niet-standaard formaten kun je aangepaste parsers implementeren via de API-webhooks om binnenkomende data te verwerken.
</Expandable>
