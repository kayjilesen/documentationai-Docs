## Overzicht

**Sections** zijn herbruikbare content-blokken die klanten zelf kunnen toevoegen, verwijderen en configureren via de theme editor. Binnen een section kunnen **blocks** gedefinieerd zijn — kleinere herhaalbare onderdelen (bijv. feature items, testimonials, FAQ-items).

<Callout kind="info" title="Belangrijk uitgangspunt">
  Alles wat klanten zelf moeten kunnen aanpassen hoort in een section met een schema — nooit hardcoded in templates. Als het niet configureerbaar is, hoort het niet in een section.
</Callout>

## Anatomie van een Section

Een section bestaat uit drie delen:

```liquid
{% comment %} sections/hero-banner.liquid {% endcomment %}

<section class="hero" style="background-color: {{ section.settings.bg_color }};">
  <div class="hero__inner">
    <h1>{{ section.settings.heading }}</h1>
    <p>{{ section.settings.subheading }}</p>

    {% if section.settings.button_label != blank %}
      <a href="{{ section.settings.button_link }}" class="btn">
        {{ section.settings.button_label }}
      </a>
    {% endif %}
  </div>
</section>

{% stylesheet %}
  .hero { padding: 4rem 0; text-align: center; }
  .hero__inner { max-width: 720px; margin: 0 auto; }
{% endstylesheet %}

{% schema %}
{
  "name": "Hero banner",
  "tag": "section",
  "class": "section-hero",
  "settings": [
    {
      "type": "text",
      "id": "heading",
      "label": "Titel",
      "default": "Welkom bij onze shop"
    },
    {
      "type": "textarea",
      "id": "subheading",
      "label": "Subtitel"
    },
    {
      "type": "color",
      "id": "bg_color",
      "label": "Achtergrondkleur",
      "default": "#f5f5f5"
    },
    {
      "type": "text",
      "id": "button_label",
      "label": "Button tekst"
    },
    {
      "type": "url",
      "id": "button_link",
      "label": "Button link"
    }
  ],
  "presets": [
    { "name": "Hero banner" }
  ]
}
{% endschema %}
```

## Stappen

<Steps>
  <Step title="Section file aanmaken" icon="file-plus">
    Maak een nieuw bestand in `sections/` aan — bijvoorbeeld `sections/featured-products.liquid`. De bestandsnaam wordt automatisch de section type (`"type": "featured-products"` in JSON templates).
  </Step>

  <Step title="Schema definiëren" icon="sliders">
    Het `{% schema %}` blok bepaalt hoe de section eruitziet in de theme editor. Belangrijkste velden:

    - `name` — zichtbare naam in de editor
    - `settings` — array van input fields (text, color, image_picker, range, select, etc.)
    - `blocks` — herhaalbare sub-items (zie volgende stap)
    - `presets` — startwaarden zodat klanten de section kunnen toevoegen via "Add section"

    Zonder `presets` is de section niet toevoegbaar — alleen bruikbaar als static section.
  </Step>

  <Step title="Blocks toevoegen voor herhaalbare content" icon="layers">
    Blocks zijn kleinere content units binnen een section. Bijvoorbeeld een FAQ section met herhaalbare vraag/antwoord blocks:

    ```json
    {
      "name": "FAQ",
      "blocks": [
        {
          "type": "question",
          "name": "Vraag",
          "settings": [
            { "type": "text", "id": "question", "label": "Vraag" },
            { "type": "richtext", "id": "answer", "label": "Antwoord" }
          ]
        }
      ],
      "max_blocks": 20,
      "presets": [
        {
          "name": "FAQ",
          "blocks": [
            { "type": "question" },
            { "type": "question" }
          ]
        }
      ]
    }
    ```

    In Liquid render je blocks met een loop:

    ```liquid
    {% for block in section.blocks %}
      <div {{ block.shopify_attributes }}>
        <h3>{{ block.settings.question }}</h3>
        <div>{{ block.settings.answer }}</div>
      </div>
    {% endfor %}
    ```

    <Callout kind="warning" title="Vergeet block.shopify_attributes niet">
      Zonder dit attribuut werkt de theme editor selection/highlight niet. Voeg het altijd toe aan het wrapper element.
    </Callout>
  </Step>

  <Step title="Dynamic sources gebruiken" icon="link">
    Laat klanten content uit hun store koppelen (producten, collecties, blog posts) via object inputs:

    ```json
    { "type": "collection", "id": "collection", "label": "Collectie" },
    { "type": "product", "id": "product", "label": "Product" },
    { "type": "blog", "id": "blog", "label": "Blog" },
    { "type": "page", "id": "page", "label": "Pagina" }
    ```

    In Liquid benader je de data direct:

    ```liquid
    {% assign collection = collections[section.settings.collection] %}
    {% for product in collection.products limit: 4 %}
      {% render 'product-card', product: product %}
    {% endfor %}
    ```
  </Step>

  <Step title="Metafields voor extra productdata" icon="database">
    Als standaard productvelden niet volstaan (bijv. ingrediënten, specificaties, CE-keurmerk), gebruik **metafields**. Configureer ze in Shopify admin onder **Settings → Custom data → Products**.

    Benader ze in Liquid:

    ```liquid
    {{ product.metafields.custom.ingredients }}
    ```

    Koppel metafields aan section settings via **dynamic sources** in de theme editor zodat klanten ze visueel kunnen koppelen.
  </Step>

  <Step title="Section in JSON template plaatsen" icon="layout">
    Voeg de section toe aan een JSON template (bijv. `templates/index.json`) of laat klanten dit zelf doen via de theme editor:

    ```json
    {
      "sections": {
        "my-hero": {
          "type": "hero-banner",
          "settings": {
            "heading": "Lente collectie 2026"
          }
        }
      },
      "order": ["my-hero"]
    }
    ```
  </Step>
</Steps>

## Veelgebruikte Setting Types

| Type | Gebruik |
|------|---------|
| `text` / `textarea` | Korte/lange tekst |
| `richtext` | WYSIWYG editor |
| `number` / `range` | Numerieke input (range = slider) |
| `checkbox` | Boolean toggle |
| `select` / `radio` | Keuze uit opties |
| `color` / `color_scheme` | Kleur of kleurenschema |
| `image_picker` | Media library afbeelding |
| `video` / `video_url` | Shopify-hosted of externe video |
| `url` | Interne of externe link |
| `font_picker` | Google Fonts / Shopify fonts |
| `html` | Raw HTML |
| `product` / `collection` / `blog` / `page` | Dynamic object references |

## Best Practices

<Callout kind="tip" title="Denk in componenten">
  Elke section moet zelfstandig werkbaar zijn. Vermijd afhankelijkheden tussen sections. Als je twee sections hebt die samen moeten werken, overweeg ze samen te voegen of via blocks op te lossen.
</Callout>

- **Minimale defaults**: vul altijd `default` in voor settings zodat presets meteen goed ogen
- **Group settings**: gebruik `header` type om gerelateerde settings visueel te groeperen
- **Scope CSS**: gebruik section class of `[data-section-id]` om styling te scopen
- **Lazy loading**: laad zware assets (video, grote images) alleen als ze boven de fold zijn
